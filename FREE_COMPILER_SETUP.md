## How It Works Behind the Scenes

The compiler uses **Piston API** (https://emkc.org/api/v2/piston/execute), which is a free, open-source code execution engine. Your code is executed securely in isolated containers.

## Browser-Based Languages

Some languages also work directly in the browser (no API calls):
- **SQL** - Uses sql.js (runs in browser)
- **Python** - Uses Pyodide (runs in browser)
- **JavaScript/TypeScript** - Runs natively in browser

## Optional: Use Judge0 Instead

If you want to use Judge0 API instead (for features like execution time and memory usage), see `JUDGE0_SETUP.md` for configuration options. But **Piston API works great by default**!

## Troubleshooting

### Code not executing?
- Make sure you're connected to the internet (Piston API requires internet)
- Check browser console for any errors
- Try refreshing the page

### Language not supported?
- Check the list above for supported languages
- Some languages might have different names (e.g., "cpp" not "c++")

### Need help?
- Check the browser console for error messages
- Make sure your code syntax is correct
- For compilation errors, check the error message in the output
